pub fn wl_surface(&self) -> Option<WlSurface>

Returns the associated wl_surface.

This will only return Some once:

  • The WL_SURFACE_SERIAL has been set on the x11 window, and
  • The wl_surface has been assigned the same serial using the xwayland shell protocol on the wayland side, and then committed.
source

pub fn wl_surface_id(&self) -> Option<u32>

👎Deprecated: Since XWayland 23.1, the recommended approach is to use [wl_surface_serial] and the xwayland shell protocol on the wayland side to match X11 windows.

Returns the associated wl_surface id, once it has been set by xwayland.

Note that XWayland will only set this if it was unable to bind the xwayland shell protocol on the wayland side.

source

pub fn wl_surface_serial(&self) -> Option<u64>

Returns the associated wl_surface serial, once it has been set by xwayland.

XWayland will set this if it has bound the xwayland shell protocol on the wayland side. Otherwise, it will set [wl_surface_id] instead.
